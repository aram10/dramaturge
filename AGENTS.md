## Project

Dramaturge is an agentic exploratory QA engine for web applications. It uses LLM-driven browser agents (Stagehand + Playwright) to autonomously explore, test, and report findings on live web apps.

**Stack**: TypeScript 5.8 · Node ≥ 20 · ES modules · pnpm 9 · Vitest 4 · ESLint 10 · Prettier · Ink/React terminal UI · Zod 4 · Playwright · Stagehand

## Setup & Commands

```bash
corepack enable
pnpm install --frozen-lockfile
pnpm run build         # clean + tsc → dist/
pnpm run test          # vitest run (600+ tests)
pnpm run lint          # eslint .
pnpm run format:check  # prettier --check .
```

Single test: `pnpm run test -- --run src/config.test.ts`

## Code Standards

### Must Follow
- Named exports only — **never** use default exports
- All relative imports use `.js` extension: `import { X } from './module.js'`
- Use `import type` for type-only imports
- Semicolons always, single quotes, 2-space indent, 100-char line width
- Trailing commas ES5 style
- `const` by default, `no-var` enforced
- No `any` — use `unknown` and type-narrow, or define proper types
- Zod schemas for config validation, discriminated unions for polymorphic types
- Options objects for functions with more than 2-3 parameters

### Testing
- Co-located test files: `foo.test.ts` beside `foo.ts`
- Use `vi.hoisted()` for mock factories, `vi.mock()` with `.js` paths
- Factory helpers for test doubles: `function makeItem(overrides = {}): Type`
- `describe()`/`it()` with descriptive names, `beforeEach(() => vi.clearAllMocks())`
- Never remove or disable existing tests

### Error Handling
- Permissive: catch errors, log them, don't crash the engine
- Workers return outcome status (`completed | blocked | timed-out | failed`)
- Browser errors auto-captured; policy layer suppresses known-noisy patterns

### Commits
- Conventional Commits: `feat:`, `fix:`, `docs:`, `chore:`, `refactor:`, `test:`, `perf:`, `ci:`
- Append `!` for breaking changes: `feat!:`
- Do not edit CHANGELOG.md (auto-generated by release-please)

## Architecture at a Glance

```
Engine Loop: Planner → Frontier Queue → Workers (Stagehand agents) → Graph Expansion → Repeat
Worker Types: navigation | form | crud | api | adversarial
Agent Modes: cua (computer-use) | dom (DOM inspection)
A2A Protocol: Coordinator + Blackboard + MessageBus → 5 agent roles (Scout, Tester, Security, Reviewer, Reporter)
```

Key entry point: `src/engine.ts` → `runEngine()`
